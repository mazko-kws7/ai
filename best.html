Python Code Snippet (Illustrative - Needs Adaptation):

import re
import os

def analyze_reports(directory="response"): # added the directory argument

    report_data = []

    # Check if the directory exists before attempting to process it
    if not os.path.exists(directory):
        print(f"Error: Directory '{directory}' not found.")
        return

    for filename in os.listdir(directory):
        if filename.endswith(".md"):
            asset_name = filename[:-3]  # Remove ".md"
            filepath = os.path.join(directory, filename)

            try:
                with open(filepath, "r") as f:
                    content = f.read()

                    # Use regex to find probability and recommendation
                    probability_match = re.search(r"Profit Probability:\s*(\d+)%", content)
                    recommendation_match = re.search(r"Recommendation:\s*(LONG|SHORT)", content, re.IGNORECASE) #Case insensitive

                    if probability_match and recommendation_match:
                        probability = int(probability_match.group(1))
                        recommendation = recommendation_match.group(1).upper()
                        report_data.append((asset_name, recommendation, probability))
                    else:
                        print(f"Warning: Could not extract data from {filename}")
            except FileNotFoundError:
                print(f"Error: File not found: {filepath}")
            except Exception as e:
                print(f"Error processing {filename}: {e}")

    # Sort by probability (highest first)
    report_data.sort(key=lambda x: x[2], reverse=True)

    # Print the results
    for asset_name, recommendation, probability in report_data:
        print(f"{asset_name}: Recommendation ({recommendation}), Profit Probability: {probability}%")

# Example usage: Assuming your reports are in a directory called "response"
analyze_reports("response")

Explanation of the Python code:

  1. analyze_reports(directory="response"):
    • Takes the directory name as an argument (defaults to “response”). This is crucial to avoid the tail: cannot open 'response/*' for reading: No such file or directory error. If your files are in the current directory, you can call the function without any arguments: analyze_reports()
  2. Directory Check:
    • os.path.exists(directory): This crucial check makes sure that the specified directory actually exists before the code tries to read files from it. If the directory does not exist, a helpful error message is printed, and the function returns, preventing the FileNotFoundError.
  3. Iteration:
    • os.listdir(directory): Gets a list of all files and directories in the given directory.
    • filename.endswith(".md"): Filters for only files ending in “.md”.
    • filepath = os.path.join(directory, filename): Creates the full path to the file.
  4. File Reading and Data Extraction:
    • try...except: Handles potential FileNotFoundError if a file is missing. Also handles other potential exceptions during file processing.
    • with open(filepath, "r") as f:: Opens the file for reading in a safe way (it will be closed automatically).
    • content = f.read(): Reads the entire file content.
    • re.search(): Uses regular expressions to find the probability and recommendation within the file content.
      • The regex r"Profit Probability:\s*(\d+)%" searches for the text “Profit Probability:”, followed by any whitespace (\s*), then captures one or more digits (\d+) before a percent sign (%). The captured digits are the probability.
      • The regex r"Recommendation:\s*(LONG|SHORT)" searches for the text “Recommendation:”, followed by any whitespace, and then captures either “LONG” or “SHORT”. The re.IGNORECASE flag makes the search case-insensitive.
    • probability = int(probability_match.group(1)): Extracts the captured probability (as a string) and converts it to an integer.
    • recommendation = recommendation_match.group(1).upper(): Extracts the recommendation and converts it to uppercase.
  5. Data Storage:
    • report_data.append((asset_name, recommendation, probability)): Stores the extracted data as a tuple in the report_data list.
  6. Error Handling:
    • print(f"Warning: Could not extract data from {filename}"): If the regex fails to find the data, a warning message is printed. Crucially, the script doesn’t crash; it continues processing other files.
    • The try...except block catches FileNotFoundError and other exceptions to prevent the script from crashing due to file issues.
  7. Sorting:
    • report_data.sort(key=lambda x: x[2], reverse=True): Sorts the report_data list in descending order based on the probability (the third element of each tuple).
  8. Output:
    • The code then iterates through the sorted report_data and prints the results in the desired format.
